Respostas reativas

2+ bikes ativas → IA pergunta sobre qual bike antes de responder

06

Controles manuais

Enviar status agora

Dispara mensagem de status atual para o cliente via WhatsApp sem esperar ele perguntar

Aprovar manualmente

Aprovação presencial: IA avisa o cliente que o adicional foi aprovado e soma o valor na OS

Pausar / Retomar IA

Toggle direto na conversa do cliente. Indicador visual no card (verde / âmbar)

Devolver para IA

Após atendimento humano, reativa a IA e limpa flags de human_takeover

Log de mensagens

Últimas 3 mensagens enviadas pela IA visíveis no modal do card

Alertas fullscreen

Aprovado, negado, cancelado, novo orçamento. Persiste via localStorage até interação.

Quando a IA age e quando para

Situação IA faz o quê? Status
Sem OS ativa, cliente fala qualquer coisa Atendimento geral — catálogo, frete, dúvidas Ativa
OS ativa, cliente manda mensagem Responde com status atual da bike Ativa
Adicional pendente Aguarda e classifica resposta do cliente Aguardando
Cliente aprova adicional Confirma, soma valor, continua informando status Ativa
Cliente nega adicional Informa que só o inicial será feito, continua status Ativa
Cliente cancela a OS inteira Para completamente. Nunca mais responde nessa conversa. Desativada
Atendente pausa manualmente Silêncio total naquela conversa Pausada
Nova OS aberta no mesmo número IA reinicia do zero para aquele cliente Reativada
OS sem número de WhatsApp Não age — apenas atendente gerencia Inativa
